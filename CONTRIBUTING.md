Contributing to Veritas

Thank you for your interest in contributing to Veritas.

Veritas is an open-source effort focused on ontological reasoning, mathematical discovery, research-to-code workflows, and evidence-backed software engineering.

Our goal is to build a system that helps transform mathematical research, algorithms, and engineering knowledge into validated, maintainable, production-grade software.

Core Principles

Contributors should align with the following principles:

Evidence over speculation
Validation over assumption
Simplicity over unnecessary complexity
Reproducibility over anecdote
Transparency over hidden behavior
Mathematical correctness over superficial optimization
Pull Requests

We welcome pull requests.

Before submitting:

Fork the repository.
Create a feature branch from main.
Add tests for new functionality.
Update documentation where appropriate.
Ensure formatting and linting pass.
Ensure all relevant tests pass.
Provide a concise explanation of the change and its motivation.

Code Style

Veritas favors:

Functional-core / imperative-shell architecture
Referential transparency where practical
Small composable functions
Higher-order functions
Explicit dependency injection
Dispatch tables over large condition chains
Strong typing
Testability and reproducibility

Research Integrity

Veritas exists to advance trustworthy mathematical reasoning and software engineering.

Contributors should:

distinguish evidence from speculation,
distinguish proofs from conjectures,
disclose limitations honestly,
avoid presenting generated outputs as verified results when validation has not occurred,
preserve reproducibility whenever practical.

Research integrity violations may be treated as Code of Conduct violations when they intentionally mislead contributors or users.

Avoid:

Hidden global state
Large monolithic functions
Implicit side effects
Premature abstraction
Magic constants
Unvalidated AI-generated behavior
Mathematical Contributions

Mathematical contributions should:

Identify assumptions explicitly
State invariants
Include evidence or proof sketches
Document failure cases
Distinguish conjectures from established results

We encourage contributors to follow the representation-first methodology documented in MATH.md.
